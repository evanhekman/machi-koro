## project
expectimax solver for solitaire Machi Koro. maximises P(win within N turns) via depth-limited search with memoisation.
solitaire variant: only BLUE + GREEN cards in supply, 3 copies each (not 6). win by building all 4 landmarks.
landmarks: train station (4c), shopping mall (10c), amusement park (16c), radio tower (22c). total = 52c.

## architecture
- `solver/` — production Rust engine. `cargo run --release -- --depth N --workers N`
- `strategy/` — Python prototyping layer. experiment here before implementing in Rust
- `web/` — FastAPI UI at localhost:3000. `just run`
- `engine/` — full multiplayer rules; only the BLUE+GREEN subset matters here
- `cache/` — zstd-compressed depth delta files, gitignored, regenerated by running solver

## what hasn't worked
- pareto frontier pruning: too many dimensions (one per card type), slower in practice
- coin capping below 52: rare to exceed it, but the extra check cost more than it saved
